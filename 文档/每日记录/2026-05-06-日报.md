# 2026-05-06 日报

## 节点传输模式多选

- 管理后台「节点管理」新增和一键部署的传输模式从单选调整为多选，支持只选 TCP、只选 XHTTP，或同时选择 TCP 与 XHTTP。
- 同时选择 TCP 与 XHTTP 时，系统按多条逻辑线路处理：TCP 和 XHTTP 分别创建独立 `nodes` 记录、独立 inbound/outbound 和独立订阅线路。
- 同一 IP 的 TCP 与 XHTTP 必须使用不同端口，默认 TCP 443、XHTTP 8443，避免一个 Xray inbound 同时承载两种 network。
- 一键部署在多传输模式下使用 `AGENT_ROLE=multi_exit`，即使只有一个出口 IP，也只安装一个 node-agent，由 `MULTI_NODE_CONFIG` 管理多条逻辑节点。
- node-agent multi_exit 配置校验从“IP 唯一”调整为“IP:Port 唯一”，允许同一 IP 分别监听 TCP 和 XHTTP。
- 新增/更新文档：规则文件、`开发方案.md`、管理接口、订阅接口、节点代理部署指南、前端页面文档。
- README 脱敏截图已刷新，`admin-nodes.png` 展示 XHTTP 逻辑节点，`admin-node-multi-ip-deploy.png` 展示 TCP/XHTTP 多选与双端口。
